# swift-lm 0.8.6

`0.8.6` is a production-readiness and correctness-evidence patch release. It
does not change the public API. The release scope is the focused
Qwen3.5, LFM2.5, Gemma4, Qwen vision, and RotorQuant gates captured in
`docs/releases/production-readiness-2026-05-05.md`.

## What 0.8.6 Changes

### Hybrid sequence-prefill correctness gate

Hybrid prefill is no longer documented or treated as disabled merely because a
model has convolution or recurrent state buffers. BF16 Qwen3.5 DeltaNet/SSM
prompt ingestion is guarded by trace equivalence against decode-equivalent
sequential ingestion, and the focused `Qwen35PromptIngestionTests` probe gate
must remain green before claiming that path.

Q3 sequence prefill remains unsupported and must continue to report an explicit
fallback reason. This release does not claim a Qwen prefill throughput win; the
Qwen3.5 benchmark note is now marked historical until the benchmark is re-run
on the current implementation.

### Release evidence and real-bundle coverage

The production-readiness evidence now records the exact focused gates, local
model bundle paths, pass/partial status, and benchmark-smoke handling used for
the release candidate. The covered release surface includes:

- Qwen3.5 real-bundle text output and BF16 hybrid prompt-ingestion trace gate.
- LFM2.5 real-bundle output and prompt-state sampling equivalence.
- Gemma4 real-bundle strict factual output.
- RotorQuant Gemma4 baseline and full K/V policy real-bundle gates.
- Qwen vision preparation and real-bundle text/image/video/mixed prompt-state
  gates.
- Text embedding runtime and real-bundle smoke checks.

`EmbeddingGemmaReferenceParityTests` remains partial because no reference
snapshot was configured. That skipped reference-parity path is not counted as
pass evidence for this release.

### Benchmark smoke sizing

Heavy benchmark suites are now documented and sized for release-smoke
validation under the repository's 120-second outer timeout rule. Oversized
suites should be split into individual test cases rather than validated by
raising the timeout.

Additional production-readiness evidence is recorded in

## Notes

- No migration is required. There are no public API changes in this release.
- Production-ready wording is scoped to the gates listed above.
- Do not claim EmbeddingGemma reference parity until a reference snapshot is
  supplied and the parity suite produces non-skip pass evidence.
- Do not claim a Qwen sequence-prefill speedup until the dedicated Qwen3.5
  benchmark is re-run on the current implementation.
