# swift-lm 0.1.0

`0.1.0` is the first public release of `swift-lm`.

## What 0.1.0 Includes

- `SwiftLM` as the public integration surface for loading and generation
- direct Metal decode and prefill execution on Apple Silicon
- HuggingFace snapshot loading from `config.json`, tokenizer files, and `.safetensors`
- regenerable `model.staf` execution cache
- DocC and repository-level usage documentation

## Supported Scope

- Apple Silicon with a usable Metal device
- Platforms currently declared in `Package.swift`: macOS 26+, iOS 26+, visionOS 26+
- Text and chat generation through `UserInput`
- Model families currently resolved by `ModelBundleLoader`:
  - Transformer families such as Llama, Qwen 2/3, Mistral, Gemma, Phi, Mixtral, DeepSeek
  - Qwen 3.5 hybrid
  - LFM2 / LFM2.5 hybrid
  - Cohere / Command-R

## Not Included in 0.1.0

- Multimodal image or video input
- Tool calling or structured function-calling APIs
- Non-Metal execution backends
- Training or fine-tuning workflows
- Broad cross-platform validation outside the package's declared Apple platform targets

## Release Validation

The `0.1.0` release candidate was validated with:

- `swift build`
- `xcodebuild build -scheme swift-lm-Package -destination 'platform=macOS,arch=arm64' CODE_SIGNING_ALLOWED=NO`
- Focused `SwiftLMTests` and `MetalCompilerTests`
- A real-model smoke path using the local `LFM2.5-1.2B-Thinking` test bundle:
  - load model bundle
  - run prompt prefill
  - decode generated tokens

## Installation

```swift
dependencies: [
    .package(url: "https://github.com/1amageek/swift-lm.git", from: "0.1.0")
]
```
