# swift-lm 0.4.0

`0.4.0` extends the renamed public API surface with clearer request-value types,
embedding entry points, and aligned user documentation.

## What 0.4.0 Includes

- `TextEmbeddingInput` as the primary public request value for embedding APIs
- `TextEmbeddingContainer.embed(_ input: TextEmbeddingInput)` as the recommended one-shot embedding entry point
- `TextEmbeddingContext.embed(_ input: TextEmbeddingInput)` for context-owned embedding execution
- `InputMessage.tool(...)` convenience initializers
- sentence-transformers metadata/runtime cleanup that models normalization as an output postprocessor
- DocC guides for:
  - container vs context responsibilities
  - text embedding usage
  - prompt-time thinking vs output-time reasoning visibility
- aligned README, developer guide, AGENTS guidance, and production-readiness documentation

## Supported Scope

- Apple Silicon with a usable Metal device
- Platforms currently declared in `Package.swift`: macOS 26+, iOS 26+, visionOS 26+
- Text and chat generation through `ModelInput`
- Text embeddings through `TextEmbeddingInput`
- Prompt snapshot reuse through `PromptSnapshot`
- Current multimodal execution support for the documented Qwen and Gemma paths

## Not Included in 0.4.0

- Tool calling or structured function-calling APIs
- Non-Metal execution backends
- Training or fine-tuning workflows
- Broad cross-platform validation outside the package's declared Apple platform targets
