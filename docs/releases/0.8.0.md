# swift-lm 0.8.0

`0.8.0` is a correctness-focused minor release that fixes real-model output
quality for Qwen3.5, LFM, and Gemma4 while expanding Metal quantization and
compiler contract coverage. The release adds probe-backed evidence tests so
future changes must prove that supported real bundles produce the expected
first tokens, not just compile or benchmark successfully.

## What 0.8.0 Changes

### Real-model output correctness

This release fixes model-family specific execution details that previously
allowed kernels to run while still producing incorrect text:

- Qwen3.5 SSM recurrence now applies the state-space norm as a float
  unit-offset RMSNorm weight, matching the HuggingFace reference behavior.
- Batched Q/K RMSNorm decode now carries and applies the fragment
  `normWeightBias`, so unit-offset norm weights survive Q/K batching.
- Gemma4 text prefill now uses prepared text embeddings and per-layer inputs
  for sequential prompt ingestion, avoiding token-ID echo behavior on direct
  text prompts.
- Gemma4 chat prompt rendering now follows the Gemma4 turn format for text,
  image, and video markers, with explicit failure for unsupported tool-use
  rendering.

### Probe-backed real output evidence

`RealOutputEvidenceTests` now records hidden/logit/output-head probes and
asserts strict output prefixes for representative real bundles:

- Qwen3.5 emits `Japan's capital is Tokyo,` for the strict capital prompt.
- LFM emits `Tokyo...` for the direct capital-completion prompt.
- Gemma4 emits `Tokyo` as the first visible token for the same factual
  completion prompt.

The existing Qwen3.5 real text test, LFM release smoke test, and Gemma4
RotorQuant real-bundle tests were tightened from loose `contains`-style checks
to prefix/token evidence where appropriate.

### Metal quantization and compiler coverage

The Metal backend gains broader quantization coverage and more explicit
contract tests:

- Quantization planning and STAF round-trip coverage expanded across supported
  affine formats.
- Unified dequant/GEMV bit-level tests now cover additional packed layouts.
- Q3 group-64 dequant-to-prefill GEMM coverage was added.
- Buffer residency, allocator behavior, prefill transfer, optimizer entry, and
  fragment binding tests were strengthened to catch routing and ownership
  regressions earlier.
- Supported quantization documentation now includes the newly covered format
  surface.

### Documentation refresh

The README was refreshed around the current public API shape, model bundle
contract, generation flow, prompt reuse, multimodal preparation, and compiler
extension model.

## Correctness

Validated before release:

```bash
perl -e 'alarm shift; exec @ARGV' 120 xcodebuild build-for-testing -scheme swift-lm-Package -destination 'platform=macOS'
perl -e 'alarm shift; exec @ARGV' 120 xcodebuild test-without-building -scheme swift-lm-Package -destination 'platform=macOS' -only-testing:'MetalCompilerTests/MetalSourceGeneratorTests/ssmRecurrenceUsesFloatUnitOffsetNormWeight()' -only-testing:'MetalCompilerTests/MetalSourceGeneratorTests/batchedQKNormDecodeAppliesWeightBias()'
perl -e 'alarm shift; exec @ARGV' 120 xcodebuild test-without-building -scheme swift-lm-Package -destination 'platform=macOS' -only-testing:'SwiftLMTests/RealOutputEvidenceTests'
```

Observed real-model outputs:

- Qwen3.5: `Japan's capital is Tokyo,`
- LFM: `Tokyo. However, the user`
- Gemma4: `Tokyo`

## Upgrade notes

- No public API migration is required for normal `ModelBundleLoader` / container
  usage.
- Existing STAF caches remain regenerable execution caches. If a cache is stale
  or format-incompatible, delete it and let swift-lm regenerate from the
  original safetensors bundle.
- Gemma4 tool-use prompt rendering is still intentionally unsupported and now
  fails explicitly.
